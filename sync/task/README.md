# 📅 Thesis Sprints & Milestones Task Logs

本目錄記錄了本論文研究開發過程中，各個重要時間節點（Sprints）的任務清單與實作焦點，展現了本研究從**理論調研、模擬引擎開發、LLM API 串接，到最終收斂指標測量**的完整演進軌跡。

## 📂 歷史任務日誌指引

### 1. 📅 4-27 任務階段 (`sync/task/4-27/`)
*   **檔案**：`task.txt`
*   **實作焦點**：
    *   **理論調研**：了解 Plackett-Luce (PL) 模型的公式與假設，並對比與 Bradley-Terry (BT) 在區組大小 k = 3, 4 時的效率差異。
    *   **噪聲分析設計**：設計兩種 Gumbel 噪聲模式（項目級與區組級），並規劃在模擬代碼中整合 PL、BT 或雙模型計算，以得出收斂速度與效率差。
    *   **小模型探索**：開始測試本機 Ollama 的串接，並規劃從導師數據中抽出 40 筆進行測試。

### 2. 📅 5-10 任務階段 (`sync/task/5-10/`)
*   **檔案**：`New Text Document.txt`
*   **實作焦點**：
    *   **模擬引擎升級**：擴展 k = 3 至 10 的模擬運行（重複次數 r = 20），並要求對照組使用統一的 Y 軸尺度。
    *   **視覺化優化**：在模擬圖表中引入 Ribbon 區間（帶狀區間）以反映多隨機種子（Multi-seed）的波動。
    *   **小模型 JSON 輸出開發**：設計 R 語言函數調用本機小模型，使其以嚴格的 JSON 格式回傳區組評估結果（含有 ID 與 solution 的鍵值對），並進行單一區組決策原因稽核。
    *   **運算速度研究**：探討 BT 與 PL 模型在理論上與實務模擬中的計算時間與開銷。

### 3. 📅 5-18 任務階段 (`sync/task/5-18/`)
*   **檔案**：`5-18 task.txt`
*   **實作焦點**：
    *   **圖表對齊**：確保所有模擬比對圖表的 Y 軸範圍一致（Y-axis alignment），以便進行跨模型、跨 k 值的精確對比。
    *   **文獻與投影片更新**：更新關於 BT 模型的說明投影片，強調其本質為成對比較（Pairwise decomposition），並補強 PL 模型的工作機制說明。
    *   **BWS 模擬**：推進 Best-Worst Scaling 的模擬實驗。

### 4. 📅 6-8 任務階段 (`sync/task/6-8/`)
*   **檔案**：`New Text Document.txt`
*   **實作焦點**：
    *   **API 串接轉移**：從本機 SLM 升級至串接雲端商業大模型 API（如 GPT 等），設置 API Key 的保護防護，並將 API Temperature 設為 0 以獲得高確定性輸出。
    *   **結構化 JSON 餵食**：研究 JSON Schema（參考 OpenAI JSON strict 與 Gemini REST），確保模型輸出的資料格式符合預期的 schema 定義。

### 5. 📅 6-23 任務階段 (`sync/task/6-23/`)
*   **檔案**：`New Text Document.txt`
*   **實作焦點**：
    *   **收斂性度量實作**：研究並實作 Rank-Biased Overlap (RBO) 指標。
